{{% alert warning %}}
Names on this page are correct for the MSP432, but lots of naming conventions are similar for all MCUs manufacturers.
{{% /alert %}}

GPIOs are used to generate _binary electric signals._

All the pins on an MCU are _grouped in **Ports**_ (GPIOPort1, GPIOPort2, ...), and almost each of them can be used as a GPIO or with an alternate function (ADC, Timer,...) which is specified by the documentation.

In general, most of the ports of an MCU are 8-bit wide, so that an 8-bit register is associated to them and it is used to select a pin function during initialization.

| Port Name                       | Function         |
| --------------------------------| ---------------- |
| P6.0 / A15 _(port 6 pin 0)_     |  GPIO or ADC     |
| P2.7 / TA0.4 _(port 2 pin 7)_   |  GPIO or timer   |
| P3.0 / UCA2STE _(port 3 pin 0)_ |  GPIO or serial  |

Like every hardware component, MCU pins have working specs. When a digital signal is passing between I/O, a current is flowing between the 2 pins. When CMOS technology is used, those currents can be $\sim 20 \,\text{nA} \div 1\,\mu\text{A}$. If those currents will go higher than a certain value (roughly some $\text{mA}$) the pins stop working.

{{< figure src="../img/gpio-pin-specs.png" title="V-I characteristic for MSP432 pins. For the MSP432, $I\_{IL} \sim (20 \,\text{nA} \div 1\,\mu\text{A})$ and $I\_{OL} \sim 6 \,\text{mA}$. _Plots are taken from TI course slides._" numbered="true" lightbox="true" >}}

$V\_{OL}$ is the maximum $\text{V}$ output that can be generated when $I\_{OL}$ is passing through the pin. {{< hl >}}If $I > I\_{OL}$ the pin stops working and can be damaged. {{< /hl >}} In general, some pins are called **high-drive pins** and they can accept higher value of max current $I\_{OL}$ (around $20 \,\text{mA}$).

## Programming I/Os

### Initialization
Before using a pin **initialization** is required. This means that a bunch of registers should be set according to the pin requested function.

Depending on the MCU, various registers can exist. For the MSP432, some of them are listed in the following:

* `P1->SEL0` select which pin of the port should be activated.
* `P1->SEL1` select if function is GPIO (0) or alternate (1).
* `P1->DIR` select pin direction (0 is input, 1 is output). Unnecessary hardware will be disabled.
* `P1->DS` set if the pin requires high drive current.
* `P1->REN` set pull-up resistors.
* `P1->IN` store the value coming into the pin.
* `P1->OUT` store the value that should be generated from the pin.

```c
/** Example taken from the TI-RSLK MAX curriculum for the MSP432 **/
// This should be run only once at startup
void SW1_Init(void) {
  P1->SEL0 &= ~0x02; // set P1.1 to active
  P1->SEL1 &= ~0x02; // set P1.1 to GPIO
  P1->DIR  &= ~0x02; // set P1.1 as input

  P1->SEL0 &= ~0x01; // set P1.0 to active
  P1->SEL1 &= ~0x01; // set P1.0 to GPIO
  P1->DIR  |=  0x01; // set P1.0 as output
}
```

### Reading GPIOs
The `IN` register should be read, but it contains the value of each pin in the port. If I am interested only in some of the values, a two-step process is required.

```c
/** Example taken from the TI-RSLK MAX curriculum for the MSP432 **/
uint8_t sw1(void) {
  uint8_t data;
  data = P1->IN;  // read all 8-bits
  data = data & 0x02; // mask with 00000010
  return data;
}
//***** or *****
uint8_t sw1(void) {
  return P1->IN & 0x02;
}
```

If the compiler is smart enough, the machine languages generated by the 2 functions (explicit and implicit) are exactly the same, so one can decide which method is easier to understand.

### Writing in GPIOs

Also, in this case, one has a 2-step process, similar to the one explained in the previous section. This time the `OUT` register is considered.

```c
/** Example taken from the TI-RSLK MAX curriculum for the MSP432 **/
void LED(uint8_t new_v) {
  uint8_t old_v;            // read current output
  old_v = P1->OUT;          // clear bit I want to change (new bit 0)
  old_v = old_v & (~0x01);  // set/clear only the new bit
  new_v = new_v | old_v;    // write on the register
  P1->OUT = new_v;
}
//***** or *****
void LED(uint8_t new_v) {
  P1->OUT = (P1->OUT & (~0x01)) | new_v;
}
```
